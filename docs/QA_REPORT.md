# Отчёт комплексного тестирования eTask

Дата проверки: 22 июля 2026 г.

## Объём проверки

Проверены REST API и матрица доступа для ролей: администратор, директор, заместитель директора, руководитель управления, эксперт. Проверка выполнялась в отдельной базе `etask_test`; рабочая база не изменялась.

Сценарии: вход/выход и JWT-cookie, блокировка пользователя, проекты и доски, задачи и подзадачи, назначение исполнителя, смена срока/приоритета/статуса, Drag & Drop API, комментарии, чек-листы, вложения, уведомления, поиск, фильтры, календарь, аналитика, сотрудники, управления, сброс пароля и аудит действий администратора.

## Исправленные проблемы

- Добавлены полные CRUD-операции проектов, досок, задач, комментариев, чек-листов и вложений.
- Добавлено безопасное удаление файлов с диска, включая вложения удаляемых подзадач.
- Некорректные UUID вложений теперь дают 404 вместо внутренней ошибки сервера.
- Убрана рассылка уведомлений на каждое обычное обновление карточки или движение Kanban; оставлены только значимые события workflow, дедлайн, комментарий, первичное назначение и закрытие.
- Добавлено редактирование задачи с учётом роли: эксперт меняет только статус; руководящие роли — рабочие поля, исполнителя и срок.
- Добавлены удаление задачи, комментария, пункта чек-листа и вложения с проверкой прав.
- Добавлено управление проектами и досками в интерфейсе.
- Фильтры Kanban расширены до проекта, управления, исполнителя, приоритета и срока; поиск учитывает основные поля задачи.
- Для администратора скрыты рабочий поиск, создание задач и уведомления; административная роль остаётся вне рабочего процесса.
- Сохранены мышиный Drag & Drop и клавиатурная альтернатива `Alt + ←/→`.
- Добавлены состояния фокуса, подписи элементов управления и адаптивные стили новых форм.
- Фирменные акценты и экран входа приведены к синей палитре eTask.

## Контроль доступа

- Администратор: управление пользователями, ролями, управлениями, паролями и журналами; доступ к задачам запрещён.
- Директор: полный рабочий доступ, проекты, доски, задачи, назначения и аналитика; техническое администрирование запрещено.
- Заместитель: полный обзор рабочего пространства, проекты, назначения и сроки; управление аккаунтами запрещено.
- Руководитель: доступ только к своему управлению, распределение и проверка задач.
- Эксперт: доступ только к назначенным ему задачам; изменение статуса, собственные комментарии и вложения.

## Итог

Автоматизированная матрица API завершилась без ошибок: **85/85**. Актуальная production-сборка успешно проходит компиляцию frontend/backend и ESLint. Критических и средних дефектов по проверенным сценариям не осталось.

Интерактивный браузерный повторный прогон актуального Docker-образа не был выполнен средой Codex: локальный адрес был заблокирован политикой браузера после исчерпания лимита локальных разрешений. Это ограничение тестовой среды, а не ошибка приложения. Для ручной приёмки используйте сценарии из `docs/ACCESS_MATRIX.md` после запуска `docker compose up -d`.

## Проверка workflow-обновления

После добавления иерархического workflow выполнены `npm run lint`, `npm run build:server`, production-сборка Docker и перезапуск `api`/`frontend`. Все контейнеры (`postgres`, `api`, `frontend`, `gateway`, `backup`) находятся в состоянии `healthy`; миграция добавила поля этапа, времени этапа, причины и счётчика возвратов, старые статусы преобразованы без удаления задач.
