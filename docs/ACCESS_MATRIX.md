# Матрица доступа eTask

Все ограничения применяются в Express API. Скрытие кнопок в интерфейсе не считается механизмом защиты.

| Ресурс / действие | Системный администратор | Директор | Заместитель | Руководитель | Эксперт |
|---|---:|---:|---:|---:|---:|
| Просмотр и изменение рабочих задач | Нет | Все | Все | Своё управление | Только назначенные ему |
| Создание проекта и доски | Нет | Да | Да | Нет | Нет |
| Создание задачи | Нет | Да | Да | В своём управлении | Нет |
| Назначение исполнителя | Нет | Да | Да | Только в своём управлении | Нет |
| Изменение статуса | Нет | Да | Да | В своём управлении | Только своей задачи |

## Жизненный цикл передачи

Задача остаётся одной записью `tasks` и последовательно проходит этапы: `Новая → Назначена → В работе → На проверке руководителя → На согласовании заместителя → На утверждении директора → Выполнена → Закрыта`.

Передача выполняется через `POST /api/tasks/:id/workflow` с действием `assign`, `submit`, `approve`, `return` или `close`. Для возврата поле `reason` обязательно. Каждая передача и возврат записываются в `task_history`, обновляют текущего исполнителя, этап, время этапа и счётчик возвратов; новый исполнитель получает уведомление.

Уведомления не рассылаются на каждое перемещение карточки или изменение обычного поля. Они создаются только для передачи/возврата, изменения дедлайна, комментария участника, первоначального назначения и окончательного закрытия задачи.

Дополнительные гарантии:

- роль, управление и активный статус перечитываются из PostgreSQL при каждом запросе;
- отключённая учётная запись немедленно теряет доступ, даже если срок JWT ещё не истёк;
- выход отзывает серверную сессию, поэтому скопированный ранее JWT перестаёт работать;
- исполнитель задачи должен относиться к управлению этой задачи;
- поиск комментариев и задач использует те же ограничения, что и обычный просмотр;
- недоступные задачи и файлы возвращаются как несуществующие, чтобы не раскрывать их наличие.
- системному администратору API задач отвечает отказом независимо от скрытия разделов в интерфейсе;
- директор не может создавать аккаунты, менять роли, сбрасывать пароли или видеть системных администраторов;
- нельзя отключить, удалить или лишить роли последнего активного системного администратора;
- административные действия, входы и выходы записываются в неизменяемый через интерфейс журнал.
